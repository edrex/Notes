Opening my laptop presents too many distractions when I’m here to write.

This is because of the many research tabs I keep around to organize active tasks. I organize tabs heirarchically, window collecting tabs relating to a particular research topic/task, then each workspace grouping windows by context (research, communications, writing, systems administration). Most tabs are only acting as a temporary bookmark for a URL, while others contain draft input or terminal sessions etc.

This has some shortcomings:

  • Pruning non-active tasks is a manual process, usually involving copying the URLs of the tabs in a given browser window to my personal wiki.
  • Consequently, I often have a backlog of tabs I Superficially, I need a way to quickly clear my desk so I can focus on writing.

Why not just close everything?

Well, I’m using open windows to keep track of multiple active research contexts.For example, among the 150 or so tabs I currently have open are windows containing research into:

  • Recreational access to isolated BLM land tracts in Southern Oregon (for hiking/MTB)
  • Readings in cryptography (foundational papers like Diffie-Hellmann, RSA, Merkle, etc)
  • Building a WebExtension for storing my browsing data to a personal server.
  • Materials for an online course in digital signal processing I am taking.

My current approach to pruning this context is to manually move the tabs into a text-based wiki organized by topic. This splits my context across two levels: short-term (tabs and windows) and long (wiki pages), which works reasonably well.

I’ve long wanted to streamline this process though.
