libinput remapping

It seems that the most well-developed tooling for this is https://gitlab.com/interception/linux/tools

It consists of intercept, which gets exclusive access to an input device and simply prints all the (raw) events to stdout, and uinput, which takes events from stdin and injects them to a given device.

Simplest example, sudo intercept -g $DEVNODE | sudo uinput -d $DEVNODE is a noop (other that introducing a small latency, TODO: measure latency)

The idea is you can write programs that transform the event stream in arbitrary ways. Example: https://gitlab.com/interception/linux/plugins/caps2esc/blob/master/caps2esc.c
